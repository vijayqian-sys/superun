## 快速流動
1. 在superun中新建一個空白項目
2. 連接 Supabase （項目 URL / API 密鑰）
3. 將需求分解為小塊,通過聊天生成計劃,確認,然後執行
4. 使用可編輯設計來調整 UI 和響應能力
5. 發布並運行回歸檢查

## Supabase 基礎知識
- 組織和項目 → 獲取 URL 和密鑰
- 啟用電子郵件登錄（您可以最初禁用驗證電子郵件）
- 設計表模式和RLS策略

## 典型頁面
- 儀表板（Columns表/統計）
- 編輯器（標題/標籤/內容區域）
- 設置（配置文件和 API 配置）

## 調試技巧
- 先調查,後修復;如果需要,回滾到工作版本
- 使用版本差異來定位更改和回歸
