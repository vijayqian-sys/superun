## 开始使用
<AccordionGroup>
    <Accordion title="superun 是什么?">
        superun 是一个人工智能驱动的平台,可以帮助任何技能水平的用户使用自然语言创建网站或应用程序.
    </Accordion>
    <Accordion title="我可以用 superun 构建什么?">
        从登陆页面到复杂的工具,内部平台和交互式原型——superun 可让您快速构建和迭代.
    </Accordion>
    <Accordion title="我需要编码经验吗?">
        不会.无论您是否编码,您都可以使用聊天和可视化编辑进行构建.
    </Accordion>
    <Accordion title="如何创建项目?">
        在仪表板消息框中描述您的目标. superun 生成一个起始版本.从小事做起,逐步完善.
    </Accordion>
    <Accordion title="我可以从模板开始吗?">
       是的.使用模板或复刻公共项目来快速启动您的构建.
    </Accordion>
    <Accordion title="注册时收不到验证码怎么办?">
        如果使用 QQ 邮箱或 Gmail 收不到验证码,请尝试以下方法:
        - 检查垃圾邮件文件夹,验证码可能被标记为垃圾邮件
        - Gmail 用户请检查 spam 文件夹,发件方是 `UCDarts`
        - 如果长时间未收到,可以重新触发注册流程
    </Accordion>
    <Accordion title="演示、原型、研发有什么区别?">
        - **演示（Demo）**:纯前端,只用来验证想法和流程,使用假数据
        - **原型（Prototype）**:等和于演示,不含研发
        - **研发（Development）**:把真实能力（如发邮件）接上去,变成真正可服务其他用户的产品
    </Accordion>
</AccordionGroup>

## 使用 superun 构建
<AccordionGroup>
    <Accordion title="如何高效构建?">
        一次专注于一项功能,逐步构建,并在修复之前尽可能地重现问题.
    </Accordion>
    <Accordion title="提示技巧">
        清晰具体;迭代;如有帮助,请包含样式或设计关键字.
    </Accordion>
    <Accordion title="后端和数据">
        使用 Supabase 实现持久性,身份验证和边缘功能.
    </Accordion>
    <Accordion title="支持哪些登录方式?">
        在您的项目中,superun 可以自动集成以下登录方式（使用 Supabase auth）:
        - **邮箱密码登录**
        - **手机验证码登录**
        - **Google 登录**

        我们正在开发基于 Supabase 的自定义微信登录功能,后续将上线并支持微信登录.
    </Accordion>
    <Accordion title="superun 提供哪些服务?">
        superun 提供以下服务,您可以将它们集成到您的项目中:
        - **superun Cloud**:将用户身份验证,资料储存以及后端功能整合到您的产品中.
        - **superun AI**:将 AI 功能整合到您的产品中.
        - **Stripe**:将付款处理功能整合到您的产品中.安全地接受付款,管理订阅并处理交易.
        - **Nano Banana**:将 AI 图像生成功能整合到您的产品中.
    </Accordion>
    <Accordion title="秘密">
        切勿在聊天中粘贴秘密.将它们存储在环境变量或秘密中.
    </Accordion>
    <Accordion title="搜索引擎优化">
       设置页面标题/描述,使用语义结构并提供社交共享图像.
    </Accordion>
</AccordionGroup>

## 特点
<AccordionGroup>
    <Accordion title="项目可见性">
        在发布设置中配置项目可见性.发布上线的项目会有几率被选择为社区展示（出现在"看看其他人的创作"中）.如果不想在社区中展示,可以联系客服去掉,或将项目下线.
    </Accordion>
    <Accordion title="如何在不发布的情况下分享项目?">
        即使项目未发布或已下线,你也可以通过内部分享链接分享给他人:
        - 在项目导航栏中找到"在新标签页中开启预览"选项
        - 点击后会在新标签页中打开预览
        - 复制该预览页面的 URL 分享给对方
        - 内部分享链接是实时的,不会出现在公开列表中
    </Accordion>
    <Accordion title="如何查看表单提交的数据?">
        可以通过以下方式查看表单提交的数据:
        - AI 在对话中会告诉查看路径
        - 需要在项目中生成后台管理页面
        - 如果使用 superun Cloud,可以在数据库中查看
    </Accordion>
    <Accordion title="咨询功能什么时候可以免费使用?">
        咨询功能在以下时机可以免费使用:
        - 在最开始生成 4 个风格之前,等待过程可以同时咨询
        - 在项目工作完成后,点击咨询后询问问题
        - 如果正在生成中,咨询需要排队,等工作完成后再咨询
    </Accordion>
    <Accordion title="模式">
       根据您的任务在可编辑设计和代码模式之间切换.注意:代码模式功能（包括查看和下载代码）需要付费用户才能使用.
    </Accordion>
    <Accordion title="我可以复制一个项目吗?">
        使用 复刻 复制项目.必须先断开与 Supabase 连接的项目.
    </Accordion>
    <Accordion title="如何发布我的项目?">
        编辑后,打开发布流程,设置可见性并生成可共享链接.
    </Accordion>
    <Accordion title="如何添加自定義域?">
        使用发布设置向导连接自定义域（支持主要 DNS 提供商）.
    </Accordion>
    <Accordion title="我可以查看已发布项目的分析吗?">
        支持基本的访问分析;您还可以集成第三方工具.
    </Accordion>
    <Accordion title="目前支持哪些文件上传格式与限制?">
        superun 目前支持的文件上传类型与限制如下:

        **支持的文件格式**
        - 图片:`png`, `jpg`, `jpeg`, `webp`, `gif`, `svg`
        - 文件与程序码／文字档:`pdf`, `js`, `ts`, `jsx`, `tsx`, `py`, `java`, `cpp`, `c`, `h`, `cs`, `php`, `rb`, `go`, `rs`, `swift`, `html`, `css`, `scss`, `less`, `json`, `xml`, `yaml`, `yml`, `md`
        - 视频:`mp4`, `avi`, `mov`, `wmv`, `flv`, `webm`, `mkv`
        - 音讯与其他二进位档:`mp3`, `wav`, `ogg`, `flac`, `m4a`, `aac`

        **大小限制**
        - 图片:单档最大 **3.5 MB**
        - 文件与程序码／文字档:单档最大 **5 MB**
        - 视频,音讯与其他二进位档:单档最大 **100 MB**
        - 每次请求最多文件数量:**10 个**
        - 请求超时时间:**60 秒**

        以上限制适用于 superun 中支持文件上传的相关流程（例如在对话中附加文件,或使用支持上传的工具时）.
    </Accordion>
    <Accordion title="我可以在提示中添加图像吗?">
        是的.粘贴草图,屏幕截图或设计参考以改进结果.
    </Accordion>
    <Accordion title="我可以将视频添加到提示中吗?">
        还没有.请改用关键帧屏幕截图.
    </Accordion>
    <Accordion title="我可以从 Figma 导入设计吗?">
        是的.从 Figma 导入作为起点,并通过聊天/视觉编辑进行完善.
    </Accordion>
    <Accordion title="我可以从现有代码（例如 GitHub）开始吗?">
        目前还没有.不支持从现有存储库启动.
    </Accordion>
    <Accordion title="如何查看我的项目历史记录?">
        使用聊天记录或历史记录面板查看,预览和恢复版本.
    </Accordion>
    <Accordion title="我可以恢复早期版本吗?">
        是的.只需单击一下即可恢复到任何先前版本,以实现安全迭代.
    </Accordion>
</AccordionGroup>

## 管理您的帐户
<AccordionGroup>
    <Accordion title="更改登录电子邮件">
        目前尚不支持.作为解决方法,创建一个新帐户并通过公开项目 + 复刻 进行迁移.
    </Accordion>
    <Accordion title="转让所有权">
       使用 复刻 或存储库协作方法来转移所有权.
    </Accordion>
    <Accordion title="我可以删除我的帐户吗?">
        联系支持人员（例如 support@uxarts.ai）以请求删除帐户.
    </Accordion>
    <Accordion title="代幣如何获取?">
        代币可以通过以下方式获取:
        - **每日签到**:每天登录自动送 10 代币,单月上限 40（无需手动签到）
        - **邀请奖励**:对方需通过你的邀请链接注册并完成一个演示（选择风格并执行完成）,完成后奖励 20 币
        - **充值**:在充值页面可以进行充值,最右边有"自定义充值"选项
        - 在代币明细中查看具体获得和消耗情况
    </Accordion>
    <Accordion title="如何聯繫技术支持?">
        正确的技术支持邮箱是 `support@uxarts.ai`（注意是 .ai 不是 .com）.你也可以通过网站上的官方支持渠道联系支持团队.
    </Accordion>
</AccordionGroup>
