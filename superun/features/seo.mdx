superun 在每个页面与元件中都内建完善的 SEO 预设.本指南将说明 superun 如何自动优化你的项目以利搜寻引擎索引,以及你如何进一步增强网站的可发现性.

## 自动 SEO 功能

superun 会在所有生成的页面上自动应用下列 SEO 优化设定:

* 保持 `<title>` 标签简洁（约 60 字元）并包含主要关键字
* 根据页面目的撰写约 160 字元的 meta description,并自然使用关键字
* 确保每个页面仅有一个 `<h1>`
* 使用语意化 HTML 结构内容
* 为图片加入具描述性与关键字相关的 alt text
* 在适用时加入 JSON-LD 结构化资料（公开的评价元件,列表页,详细页等）
* 应用图片延迟载入（lazy-load）与延后非必要脚本
* 自动生成 canonical URL
* 应用以行动装置优先的 RWD（响应式）版面与正确 viewport 设定
* 产生干净且人类可阅读的网址
* 将 SEO metadata（title,description,canonical）自动应用到 AI 生成的页面
* 对内部系统页（Analytics,Reports,Settings,Team,Integrations）自动应用 `noindex`,避免管理路径被搜寻引擎收录
* 自动加入来自 **Website Info** 设定的 Open Graph 资料（网站标题,描述,OG 图片）

这些优化设定能确保你的项目符合现代 SEO 标准,减少手动调整的需求.

## Website Info（全域 SEO 设定）

在 **Launch → SEO** 下,你可以设定整个项目的全域 metadata:

* **Website Title** — 用于 `<title>` 与 Open Graph 标签
* **Description** — 用于页面叙述与社群分享卡片
* **Image（OG Image）** — 网站在社群平台分享时显示的图片
* **Icon** — 浏览器页签与书签中显示的图示

这些设定会应用到所有生成页面,提供搜寻引擎一致且完整的 metadata.

## 生成 SEO 文件

superun 让你能直接从界面产生必要的 SEO 文件（`robots.txt` 与 `sitemap.xml`）:

1. 点击顶部导览中的 **“Launch”**
2. 在左侧选择 **“SEO”**
3. 在 **SEO File** 区块中,点击 `robots.txt` 或 `sitemap.xml` 旁的 **“Generate”**

这些文件能帮助搜寻引擎有效地爬取与索引你的项目.

### `robots.txt`
控制搜寻引擎可存取哪些页面.

### `sitemap.xml`
提供所有公开页面的完整清单供搜寻引擎索引.

两者在项目结构变动时会自动更新.

## 内建页面速度优化

效能诊断工具已移至 **Launch → Performance**.

superun 内建由 Google Lighthouse 驱动的速度审查工具,可协助找出以下方面的改善空间:

* 更快的页面渲染速度
* 更好的 SEO 评分
* 提升无障碍使用体验
* 改善用户体验

执行方式:

1. 点击 **“Launch” → Performance**
2. 查看 Lighthouse 各项指标
3. 使用 **Try to fix** 应用建议的优化措施

这可确保网站在行动与桌面装置上都维持良好效能,进而提升 SEO 与使用体验.

## Tips and Tricks

<AccordionGroup>

<Accordion title="Sitemap 加速搜索引擎索引">
Sitemap 有助搜寻引擎更快速地发现与索引你的网站内容.

1. 前往 **Launch → SEO** 并点击 `sitemap.xml` 旁的 **“Generate”**
2. 将 sitemap 提交到 Google Search Console 加速索引:
   `https://yourdomain/sitemap.xml`  
3. 当 `robots.txt` 与 `sitemap.xml` 都生成后,系统会自动在 robots.txt 中加入 sitemap 参考.

手动验证:
`curl https://yourdomain/sitemap.xml`
</Accordion>

<Accordion title="Robots.txt 搜索引擎控制设定">
`robots.txt` 用于告诉搜寻引擎网站内哪些区域可以或不可以爬取.

1. 前往 **Launch → SEO** 并点击 **“Generate”**
2. 预设设定允许所有页面被爬取,但 superun 对系统页面会自动应用 `noindex`.

范例:
```txt
User-agent: *
Allow: /
Sitemap: https://yourdomain/sitemap.xml

```
</Accordion>
<Accordion title="提升 SEO 的内部连结策略">
内部连结能帮助搜寻引擎理解网站内容结构,并提升页面权重传递效率.

superun 能协助:

- 建议相关的内部连结
- 维持合理的导航阶层
- 改善爬取深度与内容语意
</Accordion>

<Accordion title="更多优化技巧">
- 保持页面内容独特且持续更新
- 添加具有意义的图片 alt text 并压缩资产
- 针对长尾关键字进行优化
- 在适合的情况加入结构化资料
- 确保每个页面具有明确的核心主题
</Accordion>

</AccordionGroup>

## FAQ

<AccordionGroup>

<Accordion title="搜索引擎索引需要多久?">
索引通常需要数小时到数天.提交你的 `sitemap.xml` 能加速搜索引擎发现网站内容.
</Accordion>

<Accordion title="如果沒有排名怎么办?">
请检查 sitemap,内容品质,内部连结,页面速度,行动体验,并确认沒有误应用 `noindex`.
</Accordion>

<Accordion title="superun 是否支持行动装置 SEO?">
是的.所有 superun 项目皆採用行动优先的 RWD 设计并具备正确的 viewport 设定.
</Accordion>

</AccordionGroup>
