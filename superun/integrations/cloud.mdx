superun 云服务 是一个全栈云平台,负责处理应用程序的每个部分 - 从用户看到的前端到在幕后为其提供支持的后端.您所要做的就是直接在 superun 中描述您想要的内容,superun 会自动生成前端和后端,准备在云中运行.您无需管理硬件或复杂的基础设施,而是可以按需访问托管和计算,并随着应用程序的增长而自动扩展. superun 云服务 利用 Supabase 的开源基础,这意味着您从第一天起就获得了一个生产就绪的环境,而无需单独设置 Supabase.您可以使用生产就绪的数据库,实时更新和用户身份验证,无需任何手动设置步骤.

## 为什么选择superun 云服务

* **随着您的成长而扩展**:从您的第一个原型到数百万用户,superun 可根据您的需求无缝扩展.
* **默认安全**:企业级身份验证,细粒度访问控制和合规性最佳实践可保护您的应用程序和数据.

## 启用 superun 云服务

默认情况下,您的工作区启用 superun 云服务.您可以在用户首选项中管理项目的 superun 云服务 行为:**构建 → 服务 → superun 云服务. **

### 用户偏好

云集成的默认设置是**每次询问**,这意味着您决定是否为每个项目启用或禁用云.您可以随时从 **构建 → 服务 → superun 云服务 更改您的首选项. ** 当某个功能需要后端功能（例如身份验证）时,superun 会提示您.您有三个选择:
* **允许**:启用当前项目的集成.
* **拒绝**:拒绝此请求的集成（稍后可能会再次询问您）.
* **调整首选项**:更改未来项目的默认行为（不影响当前项目）.

## 配置 superun 云服务 工具

用户首选项中的每个云工具都有三个选项:
* **始终允许**:superun 自动执行操作,无需请求审核或批准.
* **每次都询问**:每当需要执行操作时,superun 都会请求您的批准.
* **绝不允许**:superun 阻止该操作,通知您需要云访问,并指示您启用云.

您可以随时从 **构建 → 服务 → superun 云服务 更改您的云工具首选项. **

#### 工具设置和默认值

|设置|描述 |默认行为 |
| ------------------- | ------------------------------------------------------------------------------------------------------------------ | ---------------- |
|云|创建并连接云项目以添加后端功能（数据库,身份验证）.需要使用云工具. |每次询问 |
|读取数据库 |允许 superun 读取您的数据库.                                                                               |始终允许 |
|阅读分析 |查询日志以调查问题和行为.                                                                     |始终允许 |
|运行安全检查 |对您的数据库运行自动安全检查.不是全面审核.                                                  |始终允许 |

|添加数据 |将数据添加到数据库表中.                                                                                  |始终允许 |
|阅读项目Info |阅读项目配置和环境详细Info.                                                                |始终允许 |
|配置身份验证 |允许 superun 配置身份验证设置（例如注册策略）.                                  |始终允许 |

## superun 云服务 功能

通过单击 **预览** **模式** 旁边的"+"按钮,您可以在任何 superun 项目中找到 **云选项卡**. superun 云服务 图标 以下功能和设置位于"云"选项卡内:

### 资料库

* **自动生成架构**:用简单的语言描述您需要的数据,superun 会为您创建表和架构.
* **无需 SQL 进行管理**:直接在 UI 中查看,编辑和组织记录.更新值,检查表并保持应用程序的数据同步,而无需编写查询.

### 用户和身份验证

* **用户管理**:查看用户帐户,跟踪活动并管理访问控制.
* **内置身份验证**:superun 可以为您的应用程序生成注册/登录页面.支持电子邮件,电话和 Google 登录.
* **默认安全**:身份验证由 superun 云服务 支持,确保用户凭据安全.

#### 开启身份验证

1. 进入 **研发 → 服务 → superun 云服务** 详情页面
2. 点击「**身份验证设定**」
3. 在「**登录方式**」中可以选择：
   - **电子邮件**: 启用邮箱密码登录
   - **电话**: 启用手机验证码登录
   - **Google**: 启用 Google 登录

启用后,superun 会自动为您的应用程序生成对应的注册/登录页面.

### 人工智能

* **内置 AI 功能**:superun 云服务 具有内置 AI,因此您可以快速创建功能,使您的应用程序更智能,更具吸引力.有关详细信息,请参阅superun AI.
* AI 支持的功能的一些示例包括:
* **人工智能摘要** – 自动将长文本压缩为清晰的要点
* **人工智能聊天机器人或代理** – 在您的应用程序中构建对话助手
* **情绪检测** – 大规模了解用户反馈
* 您可以从 AI 概览中监控项目的 superun AI 使用情况和性能.

### 纪录

* **实时监控**:实时观察应用程序活动.
* **调试和跟踪问题**:识别错误,跟踪事件并确保您的后端正常运行.

## 示例流程 - 提示 superun 构建 CRM 工具

以下是如何使用 superun 云服务 构建 CRM 工具的分步示例:

### 第 1 步:组织数据（数据库）

首先描述您需要的数据:

```
I want to build a CRM tool. I need to store:
- Companies (name, industry, website, address)
- Contacts (name, email, phone, company they work for)
- Deals (title, value, stage, contact, company)
- Notes (content, related to deals or contacts)
```

superun 自动为您创建数据库架构和表.

### 第 2 步:确保安全（用户和身份验证）

要求 superun 添加身份验证:

```
Add user authentication. Only authenticated users should be able to access the CRM.
```

superun 配置身份验证并为您的应用程序生成注册/登录页面.

### 第三步:添加人工智能超能力（AI）

使用 AI 增强您的应用程序:

```
Add AI-powered summaries for deal notes. When a user writes a note, automatically generate a summary.
```

superun 集成了 AI 功能,可使用内置 AI 自动生成摘要.
